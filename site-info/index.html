eliudnir.github.io

This whole thing started when I first got interested in using NodeJS and Pug templating as a way to “spit out” a self made static web page to host here on the github pages for my account. After many hours of “learning” (a.k.a. re-inventing the wheel) I deciced to try out some of the available generators.

One tool that seemed to hit on most of the requirements along the way was hexo, and that’s where I eventually landed. Writing a lot of scaffolding anyway for other things, I wanted this to require less of that.

It’s is a great tool to get into, their website will guide you through setting up and deploying your static site in no time. The documentation on customization & theming could definitely be improved, but it’s nothing a few hours (may feel like pulling teeth) won’t resolve if you search around a bit.

The way it works is, you install the CLI tool and create a project. You can make new posts, pages, drafts and eventually learn to customize all this to whatever needs you may have. Hexo provides you with the expected commands and tools along the way, for example: you can $ hexo generate to write fresh resources to the public directory, and $ hexo clean to get delete those same resources.

Hexo also provides you with a development server that you can run when you are drafting new things to preview on your computer before deploying to the host of your choice. Did I mention $ hexo deploy? Set up your deployment options in the base config file and deployment is laughably trivial as well.

Once you got your project established, making a new post or page is as simple as writing in markdown, which isn’t much of a curb to climb. If you’re feeling up for it, there’s no reason not to customize everything into your personal project by adding your theme folder in the themes folder in the root of the hexo project. Templates in this theme are written in EJS, but I believe hexo will autodetect your theme template filename if you wish to go with another template engine.
