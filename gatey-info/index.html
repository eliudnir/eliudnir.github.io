Gatey McGateface

Motivations / Goals

  • Write an API (in NodeJS) for property gate controller
  • Write test cases and test the API sufficiently to deploy API at the property
    • API tested on a Raspberry Pi 3
  • Write documentation and usage examples for API
    • Full install, setup and deployment documentation procedures
    • API endpoints and methods are all documented w/ usage examples in curl etc.
    • All server config variables and usage is documented
  • Write client(s)
    • Matey McGateface is the Android client that was initially requested for this project, its project info links here for now.

How it works

The API is currently run & tested on a Raspberry Pi 3, but it should run on any ARMv7/8 mini device of similar nature so long as one can get NodeJS running. The API offers an authenticated and secure way to control the property gate from a client UI. It has access level authentication and needs one (or more) administrator accounts for most user CRUD and high-level administration changes when deployed. The API asks (blocks) the deployer to create the admin account on first deployment.

Features

Server

  • Production and Debug mode (more errors/debug output and so on)
  • Debug mode server w/ file-watching (reloads)
  • Logging of gate events, errors etc.
  • Optional TLS setup in init
  • All server and API code is commented throughout

API

  • CRUD on all DB resources w/ acces rights
  • username:password(hash) and JWT-Auth
  • Promote/Demote user (admin access)
  • Keypad code auth (grant access w/ correct code)
  • Authed acces to RPi3 camera feed on client
  • Create “invite” tokens (for friends/family) with configurable options
    • N-number of attemps per token
    • GPS location on attemp (ensures proximity to gate when used)
    • Only to be used between (this time/date) and (that time/date)
    • Convert token to QR Code (can be shown to camera feed at gate)

Testing

  • All server route handlers and API features have test cases that are run with a simple command to make sure that the customer can easilly add features and run tests w/ feedback if anything breaks.

What’s being used

The server framework I use for this is HapiJS which has been great. The server and config itself is really well documented, and I have not run into any walls getting this project going.

Most of the security stuff is handled by bcrypt, NodeJS Crypto, while JWT handles TOKEN auth.

The database currently in use is a SQLite instance together with the Sequelize ORM library (making it fairly easy to change the db if one wishes to do so later).

HapiJS comes with a rich following of great plugins to help you resolve most common tasks and I use a few of these, such as Boom for HTTP error objects.

Testing is handled by Mocha & Chai assertions coupled with the request library to make HTTP requests to the API endpoints and methods available.
